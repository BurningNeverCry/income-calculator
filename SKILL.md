# 境外证券收益计算器

基于券商年度账单/交易记录文件，自动计算指定年份的境外证券收益所得。

## 工作流程

### Step 1: 确认数据来源

提示用户提供年度账单文件所在的文件夹路径。如果用户已经在之前对话中提供过路径，直接使用。

需要先识别券商类型（根据文件结构自动判断），目前已适配：
- **富途牛牛**：年度账单 .xlsx（含 证券-交易流水、证券-资金进出 等 sheet）
- **其他券商**：根据用户实际上传的文件格式，动态解析列结构

### Step 2: 扫描文件

运行扫描脚本识别年份：

```bash
python3 scripts/scan_bills.py <folder_path>
```

输出各年份对应的文件名。向用户确认扫描结果，并询问需要计算哪几年的收益。

### Step 3: 执行计算

运行计算脚本：

```bash
python3 scripts/calculate_income.py <folder_path> --target-years 2023,2024,2025
```

脚本会：
1. 按年份顺序读取所有交易流水，从最早年份开始追踪持仓成本
2. 使用加权平均成本法计算每只股票的持仓均价
3. 对目标年份的每笔卖出交易计算财产转让所得
4. 从"证券-资金进出"表提取股息收入
5. 识别期权到期作废的损失
6. 输出 JSON 格式的计算结果

### Step 4: 生成报告

将计算结果以 Canvas 可视化报告呈现，包含：
- **汇总表**：各年股息收入、财产转让所得(USD)、财产转让所得(HKD)
- **各年明细**：
  - 股息收入明细（日期、股票、币种、金额）
  - 财产转让所得 — 证券交易（日期、股票、数量、价格、净收入、均价、成本基础、盈亏）
  - 财产转让所得 — 期权交易（合约、操作、盈亏）
- **统计卡片**：关键数字一目了然

## 核心计算规则

### 加权平均成本法

| 操作 | 计算方式 |
|------|----------|
| 买入 | 新均价 = (原持仓总成本 + 本次成交金额 + 手续费) ÷ (原持仓数量 + 本次买入数量) |
| 卖出 | 成本基础 = 卖出数量 × 当前持仓均价；均价不变 |
| 清仓后重建 | 均价重新从0开始计算 |

### 代码转换处理

- DIDI → DIDIY（2022年6月退市转OTC）：成本基础自动继承
- 账户升级迁移（如美股孖展→保证金综合账户）：持仓成本不变

### 股息识别规则

从"证券-资金进出"表中，按以下条件识别股息：
- 分类含"公司行动"
- 摘要/类型含 `I/D`、`F/D`、`DIVIDEND`、`股息`
- 排除手续费行（Handling Charge / Scrip Charge / Dividend Fee）

### 期权处理

- 买入期权后到期作废 → 计入财产转让损失（损失金额 = 买入成本）
- 卖出期权获利 → 计入财产转让所得

## 已适配的文件格式

### 富途牛牛（年度账单 .xlsx）

**Sheets:**

| Sheet名 | 用途 |
|---------|------|
| 证券-交易流水 | 股票/期权买卖记录 |
| 证券-资金进出 | 股息、手续费等资金变动 |
| 证券-资产进出 | 转仓、账户升级等 |
| 账户信息 | 账户基本信息 |

**交易流水列:**

```
成交时间 | 账户名称 | 账户号码 | 品类 | 代码名称 | 交易所/市场 | 方向 | 交收日期 | 币种 | 数量/面值 | 价格 | 成交金额 | 总费用 | 变动金额
```

- `方向`：买入开仓 / 卖出平仓
- `成交金额`：负数为买入支出，正数为卖出收入
- `总费用`：手续费（始终为正数）
- `品类`：证券 / 期权

### 其他券商

遇到未适配的券商文件时，应：
1. 先读取文件结构（sheet 名、列头）
2. 识别关键字段映射：日期、股票代码、买卖方向、数量、价格、金额、手续费、币种
3. 按统一的内部格式转换后，复用加权平均成本法计算逻辑
4. 如果无法自动识别，向用户确认列含义

## 注意事项

- 文件名可能含 URL 编码字符（如 `%E5%B9%B4%E5%BA%A6`），使用 `glob.glob('*.xlsx')` 获取实际文件名
- 使用 `openpyxl` 时不要用 `read_only=True` 模式（URL编码文件名可能导致问题）
- 金额取绝对值后根据方向判断正负
- 按日期排序所有交易，确保成本计算顺序正确
